## 🛠️ 技术突破

### PowerShell 关键经验

#### 1. 变量插值陷阱
```powershell
# ❌ 错误：`$attempt/$MaxRetries` 中的 `/` 被解析为路径
Write-Host "Attempt $attempt/$MaxRetries"

# ✅ 正确：使用 `${var}` 明确变量边界
Write-Host "Attempt ${attempt}/${MaxRetries}"
```

#### 2. 参数命名冲突
```powershell
# ❌ 与 PowerShell 内置参数冲突
param([switch]$Verbose)

# ✅ 使用更具体的名称
param([switch]$DetailedOutput)
```

#### 3. 异常处理的层次
- 4xx 错误（客户端）：不重试
- 5xx 错误（服务器）：指数退避重试
- 网络错误：指数退避重试
